# DOKUMEN 2 — UI GUIDELINES
## KPI Builder untuk UMKM
### Dark Navy Futuristic Gen Z Modern Style

---

## 1. Design Direction

### Style Utama
Visual identity menggunakan gaya:

**Dark Navy Futuristic, Clean SaaS, Millennial–Gen Z Modern, Professional Consulting Tech.**

Produk harus terasa:
- Modern
- Cerdas
- Strategis
- Futuristik
- Mudah digunakan
- Tidak terlalu corporate kaku
- Tidak terlalu playful
- Cocok untuk UMKM naik kelas

### Visual Personality
Brand harus terasa seperti kombinasi antara:

1. SaaS dashboard modern.
2. Consulting performance system.
3. AI-assisted business tool.
4. Futuristic productivity app.
5. UMKM-friendly management platform.

---

## 2. Brand Color System

### Primary Color
**Dark Navy**
- Fungsi: background utama, hero section, dashboard shell.
- Karakter: stabil, tepercaya, strategis, profesional.
- Rekomendasi visual: navy gelap dengan gradient subtle.

Contoh token:
- `navy-950`
- `slate-950`
- `blue-950`

### Secondary Color
**Electric Blue / Cyan**
- Fungsi: accent, CTA, active state, glowing border, chart highlight.
- Karakter: futuristic, digital, energetic.

Contoh token:
- `cyan-400`
- `sky-400`
- `blue-500`

### Supporting Color
**Indigo / Violet**
- Fungsi: gradient, premium accent, AI feel.
- Karakter: modern, creative, strategic.

Contoh token:
- `indigo-500`
- `violet-500`

### Neutral Color
**Slate / Zinc**
- Fungsi: text, card background, border, secondary panel.
- Karakter: clean, readable, modern.

Contoh token:
- `slate-900`
- `slate-800`
- `slate-700`
- `slate-400`
- `slate-200`

## 3. Semantic Color

### KPI Status Color

#### Healthy
Makna:

Visual:
- Blue/cyan accent

## 4. Typography

### Font Direction
Gunakan sans-serif modern.

Rekomendasi:
- Inter
- Plus Jakarta Sans
- Manrope
- Satoshi-like style jika tersedia
- Untuk HTML static, gunakan font stack modern default jika tidak menggunakan external font.

### Typography Hierarchy

#### H1
Digunakan untuk headline hero.
Karakter:
- Bold
- Besar
- High contrast
- Modern

#### H2
Digunakan untuk section title.
Karakter:
- Semi-bold
- Clear
- Strategic

#### H3
Digunakan untuk card title atau dashboard panel title.
Karakter:
- Medium to semi-bold
- Ringkas

#### Body
Digunakan untuk paragraf.
Karakter:
- Readable
- Tidak terlalu kecil
- Line-height nyaman

#### Microcopy
Digunakan untuk badge, label, metric explanation.
Karakter:
- Ringkas
- Informatif
- Tidak terlalu teknis

## 9. KPI Tree Visual Guidelines

### Tujuan Visual
KPI Tree harus memperlihatkan:

1. KPI utama sebagai hasil akhir.
2. KPI driver sebagai penyebab.
3. KPI operasional sebagai pengungkit.
4. Arah hubungan sebab-akibat.
5. KPI mana yang bermasalah.
6. KPI mana yang berdampak besar.

### Struktur Visual

#### Level 0
Strategic Objective  
Contoh: “Meningkatkan Profitabilitas”

#### Level 1
Outcome KPI  
Contoh: “Net Profit Margin”

#### Level 2
Business Driver KPI  
Contoh: “Revenue Growth”, “Gross Margin”, “Cost Ratio”

#### Level 3
Operational KPI  
Contoh: “Conversion Rate”, “Repeat Purchase”, “COGS Ratio”

#### Level 4
Action Indicator  
Contoh: “Sales Follow-Up Rate”, “Stock Accuracy”

---

## 10. KPI Tree Node Design

### Node Content
Setiap node berisi:
- KPI name
- Type badge
- Actual vs target
- Status indicator
- Impact score kecil

### Node Shape
Gunakan rounded rectangle.

### Node Hierarchy
Visual hierarchy:
- Strategic Objective paling besar.
- Outcome KPI medium-large.
- Driver KPI medium.
- Operational KPI compact.
- Action indicator small.

### Node Connection
Gunakan garis/arrow yang jelas.

Arah:
- Dari driver ke outcome.
- Dari operational ke driver.
- Dari activity/action ke operational.

### Color Coding
- Healthy: green accent
- Watch: amber accent
- Critical: red accent
- Neutral: slate border
- Selected: cyan glow border

---

## 11. Cause-Effect Visualization

### Pola Visual
Gunakan format:

**Driver KPI → Impact KPI → Business Result**

Contoh:
**Repeat Purchase Rate turun → Revenue Growth melemah → Profit Margin tertekan**

### Elemen UI
1. Cause card
2. Arrow connector
3. Impact card
4. Explanation panel

### Microcopy
Gunakan bahasa sederhana:

- “KPI ini memengaruhi…”
- “Jika KPI ini turun, dampaknya adalah…”
- “Kemungkinan akar masalah…”
- “Prioritas tindakan…”

Hindari bahasa terlalu akademis.

## 12. Chart and Data Visualization Guidelines

### Chart Style
Karena fase awal menggunakan HTML + Tailwind tanpa library wajib, chart dapat berupa:
- Static visual bar
- Progress bar
- Mini sparkline pseudo element
- KPI ring placeholder
- Horizontal comparison bar
- Tree node visualization manual

### Chart Principle
- Jangan terlalu banyak grafik.
- Fokus pada interpretasi.
- Prioritaskan clarity daripada complexity.
- Gunakan label yang jelas.
- Berikan insight setelah angka.

### Recommended Dashboard Visuals
1. Progress bar achievement.
2. Status badge.
3. KPI Tree nodes.
4. Impact map.
5. Trend indicator.
6. Priority matrix mini card.

## 15. Responsive Behavior

### Desktop
- Full dashboard grid.
- KPI Tree dapat tampil horizontal.
- Sidebar/topbar terlihat lengkap.
- Metric cards dalam 4 atau 5 kolom.

### Tablet
- Dashboard grid menjadi 2 kolom.
- KPI Tree bisa horizontal-scroll.
- Insight panel turun ke bawah.

### Mobile
- Landing page single-column.
- Dashboard cards single-column.
- KPI Tree bisa menjadi vertical tree.
- Menu menjadi compact.
- KPI detail ditampilkan sebagai stacked cards.

## 16. UX Writing Guidelines

### Tone
Gunakan bahasa:
- Praktis
- Strategis
- Tidak menggurui
- Mudah dipahami owner UMKM
- Profesional tetapi tetap modern

### Contoh Copy Baik
- “Lihat KPI mana yang benar-benar mendorong profit.”
- “Temukan akar masalah sebelum mengambil keputusan.”
- “Hubungkan target bisnis dengan aktivitas harian tim.”
- “Jangan hanya pantau angka. Pahami penyebabnya.”

### Hindari
- Bahasa terlalu teknis.
- Istilah enterprise yang terlalu berat.
- Copy yang terlalu panjang.
- Klaim berlebihan seperti “pasti profit naik”.

---

## 17. Accessibility Guidelines

Frontend harus memperhatikan:

1. Kontras teks cukup tinggi.
2. Ukuran font body mudah dibaca.
3. CTA terlihat jelas.
4. Status KPI tidak hanya mengandalkan warna.
5. Badge harus memiliki teks.
6. Mobile layout tidak terlalu padat.
7. Navigasi mudah dipahami.
8. Button memiliki label yang jelas.

---

## 18. SOT UI Tokens

### Border Radius
- Small: untuk badge
- Medium: untuk input/filter
- Large: untuk cards
- Extra large: untuk hero panels

### Shadow
- Gunakan shadow halus.
- Hindari shadow berlebihan.
- Gunakan glow hanya untuk accent.

### Border
- Gunakan border tipis slate.
- Selected element menggunakan cyan border/glow.

### Background
- Body: dark navy
- Panel: dark slate
- Elevated card: slate slightly lighter
- CTA: cyan/blue gradient

## 19. Visual Quality Bar

UI dianggap memenuhi standar jika:

1. Terlihat seperti SaaS modern.
2. Dashboard tidak terasa seperti template admin generik.
3. KPI Tree mudah dimengerti.
4. Landing page mampu menjual value produk.
5. Warna navy dan cyan konsisten.
6. Tampilan cocok untuk Gen Z/millennial tetapi tetap dipercaya owner bisnis.
7. Tidak ada elemen visual yang terlalu ramai.
8. Setiap panel memiliki fungsi jelas.
